Semantic search engine

Python · NLP · ESA · BERT · Source on GitHub

Academic IR project on Cranfield: explicit semantic analysis (ESA) over Wikipedia-backed vectors, normalization, and BERT spell correction (nDCG@3 0.234).

This was a semantic search system built on the classic Cranfield benchmark collection, aimed at ranking short relevance judgments more like a human would than a literal keyword engine.

Documents and queries were represented with Explicit Semantic Analysis (ESA): terms expand into weighted concept vectors backed by Wikipedia, so related wording still overlaps in concept space even when the strings differ. I applied consistent text normalization before indexing so punctuation and casing did not fragment the vocabulary.

Query preprocessing included spell correction using BERT masked language modeling, which helped noisy or mistyped queries map closer to the right concepts. After retrieval and ranking, we measured nDCG@3 at 0.234 on Cranfield, reasonable for a lightweight academic pipeline without heavy neural reranking.

The runnable pipeline (Cranfield I/O, preprocessing stages, retrieval driver, and evaluation hooks) is in navnitan-7/toy_search_engine (main.py orchestrates stages; module files match each preprocessing and IR step).

Detailed report

How the system is wired end-to-end: lexical preprocessing from the repo, semantic representations and spell correction on top, then retrieval metrics.

Problem and benchmark

Cranfield is a small but standard test collection for ad-hoc retrieval: a fixed document set, topics with short queries, and binary relevance labels. The goal is to rank documents so judged-relevant items appear early, measured with metrics such as nDCG at cutoff 3.

Semantic layer

Lexical overlap misses paraphrases and related concepts. ESA maps each term to a sparse vector over Wikipedia articles (or similar concepts), then document and query vectors are built by aggregating those concept weights. Similar meaning yields overlapping mass even when surface strings differ. Normalization before that step keeps token boundaries stable so the concept index is not polluted by trivial variants.

For queries that arrive with typos or keyboard noise, BERT masked language modeling suggests corrections before concept expansion, which reduces concept drift on malformed input.

Results

On Cranfield, the pipeline achieved nDCG@3 of 0.234, useful as a course-scale system without a large cross-encoder reranker. The main levers were cleaner text going into ESA and better query strings after spell correction.
