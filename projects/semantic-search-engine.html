Semantic search engine

Python · NLP · ESA · BERT

Academic IR project on Cranfield: explicit semantic analysis (ESA) over Wikipedia-backed vectors, normalization, and BERT spell correction—nDCG@3 0.234.

This was a semantic search system built on the classic Cranfield benchmark collection, aimed at ranking short relevance judgments more like a human would than a literal keyword engine.

Documents and queries were represented with Explicit Semantic Analysis (ESA): terms expand into weighted concept vectors backed by Wikipedia, so related wording still overlaps in concept space even when the strings differ. I applied consistent text normalization before indexing so punctuation and casing did not fragment the vocabulary.

Query preprocessing included spell correction using BERT masked language modeling, which helped noisy or mistyped queries map closer to the right concepts. After retrieval and ranking, we measured nDCG@3 at 0.234 on Cranfield—reasonable for a lightweight academic pipeline without heavy neural reranking.
