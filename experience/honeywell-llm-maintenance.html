Honeywell · Backend Engineer · Jul 2022 to Nov 2024

LLM Based Maintenance Assist

Pgvector · Azure Document Intelligence · Azure Search

I designed an ingestion and retrieval stack that pulled maintenance documents from many formats and systems into a single searchable layer, using Azure Document Intelligence, embeddings in pgvector, and semantic search so a maintenance assistant could answer with grounded context. The pipeline was modular: parsers and embedders plug in, so we could swap models without rewriting the core flow.

Maintenance knowledge was scattered across PDFs, scans, legacy CMMS exports, and vendor bulletins. I built an ingestion framework that normalizes those heterogeneous sources: layout-aware extraction with Azure Document Intelligence, chunking tuned for procedure boundaries, and metadata that preserved document type and equipment tags for filtering.

For retrieval we combined vector search (embeddings stored in Postgres with pgvector) with semantic ranking in Azure Cognitive Search so technicians got passages that matched intent, not just keywords. That materially improved precision and recall for chatbot-style queries compared with our earlier lexical-only baseline.

The architecture was deliberately pluggable: parsers, embedders, and the search backend communicate through small interfaces, which made it practical to trial new embedding models or routing strategies without a rewrite. That mattered because the LLM landscape was moving quickly and we needed upgrade paths without stopping shipping features.
